# root-plans 目录分层说明

本文档说明 `development/latest-dev-docs/root-plans` 下 A/B/C/D 四层的职责与使用方式。

## A 层：`A_INDEX/`

- 作用：入口与导航层，给出当前可用文档、索引与阅读顺序。
- 典型内容：`index.md`（快速索引）、`README.md`（总览）、`DIR_MAP.md`（分层说明）。
- 面向对象：任何首次进入该目录的协作者。

## B 层：`B_MERGED/`

- 作用：合并后的主方案层，沉淀统一版本与执行口径。
- 典型内容：`MERGED_PLAN.md`。
- 使用建议：默认优先阅读本层获取当前主计划。

## C 层：`C_REVIEW/`

- 作用：评审与校对层，记录保留/删除/疑似过时项与校验结果。
- 典型内容：`MERGED_PLAN_REVIEW.md`。
- 使用建议：当需要追溯“为什么这样合并”时，回看本层。

## D 层：`D_SOURCE/`

- 作用：来源材料层，保存输入文档与原始上下文。
- 典型内容：状态报告、任务看板、证据矩阵、发布说明等源文档。
- 使用建议：当主方案存在疑问时，用本层做来源核对。

## 推荐阅读路径

`A_INDEX` -> `B_MERGED` -> `C_REVIEW` -> `D_SOURCE`

该顺序可在保证阅读效率的同时保留可追溯性。
